Введение

Gotcha — универсальный инструмент для тестирования сетевой безопасности. Он позволяет выполнять различные виды атак (флуды, подмены, истощение ресурсов), а также проводить диагностику сети. Программа предназначена для специалистов по информационной безопасности, системных администраторов и студентов, изучающих кибербезопасность.

⚠️ Важно: Использовать Gotcha разрешено только на собственных сетях или при наличии письменного разрешения владельца. Любое несанкционированное применение может являться уголовным преступлением.

Установка и требования

Системные требования

  • Windows 10 21H2 / Windows 11
  • Процессор: 2 ядра (Тесты проводились на AMD A10-5800K)
  • ОЗУ: от 4 ГБ
  • Сетевой адаптер

Обязательное программное обеспечение

  • Npcap — драйвер для захвата и генерации пакетов. Скачайте с npcap.com. При установке обязательно выберите опцию WinPcap API-compatible Mode.
  • Wireshark (рекомендуется) — для анализа трафика и отладки. Скачайте с wireshark.com
  • Nmap — для нахождения уязвимостей. Скачайте с nmap.com
  • TCP optimizer — для оптимизации сетевой карты ( возрастет кол-во создаваемых пакетов для dos атаки ). Скачайте с speedguide.net
  • Microsoft Visual C++ Redistributable (последняя версия).

Установка Gotcha

  • Запустите установщик от Имени Админстратора, после пройдите процедуру установки зависимостей ( Обязательного ПО )

Интерфейс программы

Главное окно разделено на вкладки: «Вспомогательное» (внутри «Доступ» и «Настройки») и «Атаки» (все инструменты тестирования). Внизу отображаются статус, загрузка CPU и RAM. Каждая вкладка имеет свои поля, кнопки управления и окно лога.

Вкладка «Доступ» — диагностика сети

Набор утилит для первичного анализа сети и сбора информации о цели.

  • ICMP Ping — отправляет 4 эхо-запроса. Позволяет проверить доступность узла и измерить RTT.
  • Port Scan — сканирует список популярных TCP-портов (21,22,23,25,53,80,110,143,443,993,995,3389). Выводит открытые порты.
  • Traceroute — трассировка маршрута до указанного IP/домена.
  • Таблица маршрутизации — показывает IPv4 маршруты.
  • Сетевые адаптеры — список интерфейсов, их IP и MAC.
  • Сканировать сеть — ARP-сканирование подсети /24. Выводит IP, MAC и hostname всех активных устройств. Полезно перед атакой для определения шлюза и жертвы.

Перехват пакетов

Позволяет захватывать, анализировать, редактировать и пересылать пакеты.

Основные возможности

  • Фильтр BPF — можно задать условие (например, tcp port 80 или icmp).
  • Автоматические ответы — на каждый перехваченный пакет отправляется заданное количество ответов (например, ICMP Echo Reply на ping).
  • Редактор пакетов — выберите пакет из списка, нажмите «Редактировать». Можно изменить MAC, IP, порты, TCP-флаги, payload (в hex или текстом). После редактирования пакет можно отправить в сеть.
  • Ручная отправка — захваченный или отредактированный пакет можно отправить указанное количество копий.

Этот инструмент помогает тестировать межсетевые экраны, системы обнаружения вторжений (IDS) и поведение сетевых устройств при нестандартных пакетах.

DHCP Starvation (истощение пула DHCP)

Атака направлена на исчерпание пула IP-адресов DHCP-сервера. Программа генерирует множество запросов с уникальными MAC-адресами, заставляя сервер резервировать адреса. Легитимные клиенты не смогут получить IP.

Параметры

  • Размер пула — сколько уникальных MAC будет использовано (обычно 200–500).
  • Кол-во запросов — общее количество попыток.
  • Задержка — пауза между запросами (сек).
  • Таймаут Offer/ACK — время ожидания ответа от DHCP-сервера.

В логе отображаются полученные IP (OFFER) и подтверждённые (ACK). Захваченные адреса сохраняются в статистике. Атака полезна для проверки надёжности DHCP-инфраструктуры.

DoS-атаки (флуд)

Высокоинтенсивная отправка пакетов заданного протокола. Поддерживаются TCP, UDP, ICMP, ARP, DNS.

Общие параметры

  • IP адрес — цель.
  • Порт (для TCP/UDP).
  • Размер пакета (байт) — для TCP, UDP, ICMP. Для ARP и DNS размер фиксированный.
  • Время (сек) — длительность атаки. 0 = бесконечно (остановка кнопкой).
  • Интерфейс — через который отправлять.
  • Случайный IP / MAC — подмена адреса источника для имитации распределённой атаки.

Для каждого протокола используется отдельный оптимизированный модуль. В реальном времени отображаются количество отправленных пакетов и скорость (pps). Атаки позволяют оценить устойчивость серверов и сетевого оборудования к флуду.

💡 Совет: для TCP-флуда используйте случайный IP и MAC — это усложнит фильтрацию на стороне жертвы. Для ARP-флуда указывайте широковещательный MAC (ff:ff:ff:ff:ff:ff) для максимального эффекта.

ARP Spoofing (подмена ARP)

Классическая атака «человек посередине» на канальном уровне. Атакующий отправляет поддельные ARP-ответы, убеждая жертву и шлюз, что его MAC-адрес принадлежит другому узлу. Весь трафик между жертвой и шлюзом начинает проходить через атакующего.

Настройка

  • IP цели — атакуемый хост (например, 192.168.1.10).
  • IP шлюза — основной шлюз сети (192.168.1.1).
  • Интервал (сек) — частота отправки поддельных ARP (2–5 сек).
  • Восстанавливать ARP после остановки — при отключении атаки отправляются корректные ARP-пакеты для восстановления нормальной работы сети.

Для корректной работы атакующая машина должна быть включена в ту же сеть, что и жертва, и иметь включённую пересылку пакетов (IP forwarding). Программа автоматически пытается включить службу RemoteAccess.

Применение: перехват сессий, прослушивание трафика, атаки на нешифрованные протоколы (HTTP, FTP, Telnet).

DNS Spoofing (подмена DNS)

Перехватывает DNS-запросы и подменяет ответы. Позволяет перенаправлять пользователей на подложные сайты.

Возможности

  • Задание правил подмены в формате домен → IP. Поддерживаются маски (например, *.google.com).
  • Режим catch-all — подменять все запросы на указанный IP.
  • TTL ответа (сек) — время жизни записи в кэше жертвы (рекомендуется 1–5 секунд).

При перехвате в лог выводится запрошенный домен и подставленный IP. DNS Spoofing используется для фишинга, обхода блокировок, тестирования DNS-фильтров.

Примечания по использованию DNS spoofing

1. DNS сервер не должен быть одновременно шлюзом в сети к которой подключена жертва

2. При запуске DNS spoofing убедиться что работает ARP spoofing иначе вы не увидите DNS запросы от жертвы

3. Убедиться что IPv6 ответы от реального сервера не приходят раньше чем ваши ( Попробуйте отключить IPv6 у жертвы )

MAC flood (заполнение CAM-таблицы)

Атака на коммутаторы. Отправляется множество Ethernet-кадров с разными MAC-адресами источника. CAM-таблица коммутатора переполняется, и устройство начинает работать как хаб (ретранслировать трафик во все порты).

  • MAC назначения — обычно широковещательный (ff:ff:ff:ff:ff:ff) или MAC шлюза.
  • Случайный MAC источника — каждый кадр получает новый случайный MAC (до 65 тыс. уникальных).
  • Время атаки — рекомендуется запускать на 30–60 секунд, чтобы перегрузить таблицу.

Эффективность зависит от модели коммутатора (размера CAM-таблицы). Атака помогает проверить защиту от MAC-флуда (функции типа port security).

Как повысить скорость отправки пакетов (PPS)

Если вы наблюдаете низкую производительность (менее 50 000 пакетов/с), выполните следующие шаги:

  1. Используйте SG TCP Optimizer — скачайте бесплатную утилиту, запустите от администратора, выберите скорость вашего интерфейса (100+Mbps), нажмите «Optimal», затем «Apply changes» и перезагрузитесь. Это оптимизирует параметры TCP/IP и автонастройку окна.
  2. Отключите антивирус и брандмауэр на время тестов — они могут инспектировать и ограничивать сырой трафик.
  3. Закройте фоновые приложения, использующие сеть (торренты, облачные синхронизации).
  4. Установите приоритет процесса «Высокий» через диспетчер задач.
  5. В настройках сетевой карты (Диспетчер устройств → Свойства → Дополнительно) отключите:
    • Interrupt Moderation
    • Flow Control
    • Large Send Offload (LSO)
    • Checksum Offload (для экспериментов)
    Установите Receive Buffers и Transmit Buffers в максимальное значение.
  6. Используйте проводное соединение — Wi-Fi вносит дополнительные задержки и потери.

После этих действий скорость отправки может достигать 200–400 тысяч пакетов в секунду на современном оборудовании.

Правовые и этические аспекты

⚖️ Важно: Gotcha предназначена исключительно для тестирования собственных сетей и образовательных целей. Несанкционированные атаки на чужие сети являются нарушением законодательства большинства стран (в том числе ст. 272–274 УК РФ, Computer Fraud and Abuse Act в США и др.). Автор не несёт ответственности за любые последствия неправомерного использования.

При работе с инструментом всегда следуйте этическим нормам: получайте письменное разрешение владельца, тестируйте в изолированных лабораторных средах, не причиняйте реального вреда продуктивным системам.

Часто задаваемые вопросы (FAQ)

ВопросОтвет
Почему при запуске пишет «No Ethernet interface found»?Убедитесь, что сетевой адаптер включён и имеет IP-адрес. Попробуйте выбрать интерфейс вручную из выпадающего списка.
Как узнать MAC-адрес шлюза или жертвы?Используйте кнопку «Сканировать сеть» во вкладке «Доступ» или запустите ARP-запрос вручную через командную строку: arp -a.
Влияет ли атака на производительность моего компьютера?Да, особенно при высокой скорости (от 100k pps). Рекомендуется запускать на выделенной машине или в виртуальной среде с прямым доступом к сетевому адаптеру.
Можно ли остановить атаку досрочно?Да, для каждой атаки есть кнопка «Остановить». В бесконечном режиме достаточно нажать эту кнопку.
Почему DHCP Starvation не захватывает IP?Возможно, DHCP-сервер настроен на защиту от истощения (ограничение количества запросов от одного MAC, или используется авторизация). Уменьшите скорость запросов (увеличьте задержку) или используйте другой сегмент сети.
Как обновить Gotcha?Скачайте новую версию с официального источника и замените исполняемые файлы. Настройки не сохраняются между версиями.
Почему не найден .exe файл?Проверьте папку bin рядом с программой. Если файлы отсутствуют, скачайте полную версию или скопируйте их из архива. Не переименовывайте исполняемые файлы.
Ошибка открытия интерфейса / pcap_openNpcap не установлен или установлен без режима совместимости. Переустановите Npcap с опцией «WinPcap API-compatible Mode». Также проверьте, что интерфейс активен (подключён кабель, включён Wi-Fi).
Низкая скорость отправки пакетов (PPS)Откройте SG TCP Optimizer (бесплатная утилита). Установите слайдер в положение 100+ Mbps, выберите Optimal, затем примените. После перезагрузки скорость должна вырасти. Также отключите антивирус и брандмауэр на время тестов.
ARP Spoofing не работает / жертва теряет связьУбедитесь, что в системе включён IP forwarding. Откройте PowerShell от администратора и выполните:
Set-Service RemoteAccess -StartupType Automatic
Start-Service RemoteAccess
Также проверьте, что выбран правильный сетевой интерфейс.
Программа зависает или не отвечаетЗавершите процесс через диспетчер задач, затем перезапустите от администратора. Убедитесь, что драйвер Npcap не конфликтует с другими сетевыми анализаторами.
